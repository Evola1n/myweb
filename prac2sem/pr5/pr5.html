Async та Fetch API - Питання та відповіді

1. Які переваги має async/await над ланцюжками .then()?
`async/await` робить код більш читабельним, зменшує вкладеність і спрощує обробку помилок через `try/catch`.
2. Як Promise.all може покращити паралельну обробку запитів?
`Promise.all` дозволяє виконувати кілька асинхронних операцій одночасно та отримати всі результати разом, що підвищує продуктивність.
3. Які типові помилки можуть виникнути при використанні Fetch API?
Можливі помилки: невірний URL, проблеми з мережею, неправильний формат відповіді або відсутність перевірки `response.ok`.
4. Як перевіряти статус відповіді у Fetch API і чому це важливо?
Необхідно перевіряти `response.ok`, щоб переконатися, що запит виконаний успішно. В іншому випадку слід обробити помилку (`response.status`).
5. Які ризики пов'язані з необробленими помилками в асинхронному коді?
Необроблені помилки можуть спричинити некоректну роботу програми, зупинку виконання коду або відсутність реакції на збої API.
